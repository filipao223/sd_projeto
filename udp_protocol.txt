Exemplo de um datagrama
[ feature |  2  ;   username  |  flipao   ;   acao  |  renomear_musica_nr3 ]

/*------Todos------*/
Todos os datagramas têm este campo

--> feature : string  (Depois cast para integer)
    feature:
        login (1) - (feature_LOGIN)
        manage (2) - (feature_MANAGE), ....
        search
        details
        critique
        make_editor
        note_editor
        note_new_edit
        note_deliver
        upload
        share
        download
        callback

login

--> username : string
--> password : string
<-- resposta : callback

logout

--> username : string

gerir

--> username : string
--> acao : string
<-- resposta : callback

pesquisa 

--> tipo : int
--> dados : string
<-- resposta : callback

consulta album 

--> nome : string
<-- resposta : callback

consulta artista 

--> nome : string
<-- resposta : callback

critica

--> user : string
--> nome album : string
<-- resposta : callback

privilegios

--> nome do editor : string
--> nome do utilizador : string
<-- resposta : callback

notificacao editor

<-- nome novo editor : string

notificao alteracao album

<-- nome editor : string
<-- nome album : string

notificao alteracao artista

<-- nome editor : string
<-- antigo nome artista : string
<-- novo nome artista  : string


entrega notificacoes 

<-- username : string
<-- notificacoes : Array de string

transferencia de musicas para o server

--> user : string
--> nome musica : string
--> ficheiro musical : object
<-- resposta : callback   (pode receber n so um boolean mas tb p .ex o url da musica)

partilha 

--> nome utilizadores : Array de string
--> nome musica : string
<-- resposta : callback

transferencia de musica para utilizador

--> user : string
--> musica : string
<-- resposta : callback

callback

<-- feature : string   (feature pedida ao servidor, o pacote contem na mesma o feature no inicio [Callback])
<-- username : string  (user que pediu)
<-- resposta : string
<-- opcional : object   (Em algumas features, como o login, será null, noutras terá alguma coisa, como uma musica p.ex)
